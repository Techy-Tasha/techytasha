Command Reference

  • pwd - print working directory
  • ls - list everything inside the current directory
  • ls -l - lists directory vertically, and alphabetically, also shows permissions, date and file creator, and more.
  • cd ~ - takes me back to the home directory
  • chmod - adjusts permissions
  • mkdir - make a new directory
  • touch - create a new file
  • rm - delete file
  • rm-r - remove directory and all files
  • mv - move
  • cp - copy

Now, I was finally able to move the files from my local device to Github.

  1. created the repo in github with .readme
  2. cloned repo to local directory (git clone xyz)
  3. made changes in local directory
  4. pushed changes to Github
    • committed local code to GitHub through the command line. did this by using:
      1. git init - b main (Initialize the local directory as a Git repository.)
      2. git add . (gets files ready to move)
      3. git commit -m “first commit” (adding files to local git)
      4. git push origin main (pushes files to github)
